# reflection.md – Lessons Learned in Template Selection and Kanban Customisation

## Rural Hospital Digital Decision Support System

---

## Introduction

This assignment provided practical experience in applying Agile project management using GitHub Projects. While Kanban boards and Agile principles were understood theoretically, implementing them in a real project revealed several challenges related to decision-making, workflow design, and tool limitations.

---

## Challenge 1: Selecting the Appropriate Template

One of the main challenges was choosing the most suitable GitHub project template. Initially, the Automated Kanban template seemed ideal due to its automation features. However, further analysis showed that it lacked structured sprint support and would require significant customisation.

The Team Planning template was ultimately selected because it provides built-in support for iterations, structured workflows, and custom fields. This made it better aligned with the Agile planning work completed in previous assignments.

---

## Challenge 2: Designing a Realistic Workflow

Another challenge was designing a Kanban workflow that accurately reflects real software development processes. The standard columns were not sufficient, which led to the addition of "Testing" and "Blocked" columns.

This required thinking about how tasks progress in real-world systems, especially in healthcare environments where validation and dependencies are critical. The process highlighted the importance of adapting tools rather than using them in their default form.

---

## Challenge 3: Applying Work-In-Progress Limits

Applying WIP limits was challenging, particularly as a solo developer. It required discipline to avoid working on multiple tasks simultaneously.

Limiting tasks in progress improved focus and helped ensure that tasks were completed fully before moving on to new work. This reinforced the importance of managing workflow efficiently rather than simply completing tasks.

---

## Comparison with Other Tools

GitHub Projects offers strong integration with repositories, making it ideal for development-focused projects. Compared to Trello, it provides better traceability between tasks and code, although Trello is more visually intuitive.

Compared to Jira, GitHub Projects is simpler but lacks advanced features such as reporting and analytics. However, for this system, its integration and simplicity make it an effective choice.

---

## Conclusion

This assignment demonstrated that effective Agile project management requires more than just using tools. It involves making informed decisions, designing workflows carefully, and continuously adapting processes to suit the system’s needs.

The experience improved my understanding of Kanban boards, Agile workflows, and the importance of maintaining an up-to-date project board throughout development.

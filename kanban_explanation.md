# kanban_explanation.md – Kanban Board Definition and Purpose

## Rural Hospital Digital Decision Support System

---

## 1. What is a Kanban Board?

A Kanban board is a visual project management tool used to track tasks as they move through different stages of a workflow. Each task is represented as a card, and columns represent stages such as planning, development, testing, and completion.

---

## 2. System Kanban Board — Column Structure

| Column | Purpose |
|---|---|
| Backlog | Contains all identified tasks and user stories |
| Ready | Tasks prepared and ready for development |
| In Progress | Tasks currently being developed |
| Blocked | Tasks that cannot proceed due to dependencies |
| Testing | Tasks being validated against requirements |
| Done | Completed and verified tasks |

---

## 3. Workflow Visualisation

The board visualises workflow by showing how tasks move from the Backlog to completion. Each stage reflects progress in system development, making it easy to identify delays and track work status.

---

## 4. Work-In-Progress (WIP) Limits

To prevent overload and improve efficiency, WIP limits are applied:

- In Progress: Maximum 3 tasks  
- Testing: Maximum 3 tasks  
- Blocked: Maximum 3 tasks  

These limits ensure that tasks are completed before new ones are started.

---

## 5. Agile Alignment

The Kanban board supports Agile principles as follows:

| Agile Principle | Implementation |
|---|---|
| Continuous delivery | Tasks are completed incrementally |
| Adaptability | Tasks can be reprioritised easily |
| Transparency | All tasks are visible on the board |
| Efficiency | WIP limits reduce multitasking |

---

## 6. Conclusion

The Kanban board improves workflow visibility, ensures efficient task management, and supports Agile development for the Rural Hospital Digital Decision Support System.

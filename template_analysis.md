# Template Analysis and Selection

## Comparison of GitHub Project Templates

| Template Name     | Columns / Workflow                          | Automation Features                                      | Suitability for Agile Methodologies |
|------------------|----------------------------------------------|----------------------------------------------------------|------------------------------------|
| Basic Kanban      | To Do → In Progress → Done                  | No automation; tasks moved manually                      | Suitable for simple workflows but limited Agile support |
| Automated Kanban  | To Do → In Progress → Done                  | Automatically moves issues when assigned or closed       | Highly suitable for Agile development |
| Bug Triage        | New → Investigating → Fixed                 | Automatically organizes issues                           | Suitable for bug tracking, not full Agile workflow |
| Team Planning     | Backlog → Ready → In Progress               | Supports workflow organization                           | Suitable for Agile planning |

---

## Template Selection Justification

The Automated Kanban template was selected because it supports Agile workflows through built-in automation. 
It automatically moves issues across columns when their status changes, reducing manual effort.

This is useful for managing user stories and sprint tasks from previous assignments. 
It also improves workflow visibility and aligns with Agile principles such as continuous delivery and adaptability.
